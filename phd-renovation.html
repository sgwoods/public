PhD Renovation: Program Understanding via Constraint Satisfaction

This project tracks the renovation of Steven Woods' PhD research code on automated program understanding via constraint satisfaction, originally developed at the University of Waterloo and now modernized for SBCL on current hardware.

The repository combines restored historical snapshots, a runnable modern baseline, replicated experiment batches, and supporting documentation for understanding how the thesis-era systems behaved and how faithfully they can be reproduced today.

Public project note: active empirical and archival work is underway. As of March 16, 2026, the repository includes modern SBCL runs, preserved historical snapshots, and refreshed data assets for the original ADT situation experiments.

Current State

  • The primary systems include MAP-CSP (`adt`), Memory-CSP (`memory-search`), and Quilici-style search variants.
  • The codebase is organized around runnable SBCL entry points, FiveAM test suites, historical snapshots, and archived experiment outputs.
  • The repository reports that SBCL experiment results on modern hardware statistically match the original Allegro CL/SPARC results across the four maintained code snapshots.
  • Current repository work includes preserving thesis-era data assets in directly usable form rather than relying on obsolete compressed archives.

Renovation Priorities

  1. Preserve the historical CSP, memory-based search, and Quilici-search implementations in runnable form.
  2. Keep the thesis-era datasets, outputs, and experiment structure accessible on modern systems.
  3. Document what matches the original SPARC/Allegro behavior and what differs under SBCL or cleaned-up workflows.
  4. Maintain a clear bridge between the historical research artifacts and the modernized reproducible codebase.

Repository Highlights

  • Multiple maintained system trees: `csp/`, `qcsp3/`, and preserved 1996/1997 snapshots.
  • Replicated experiment batches under `Q-Batch-SBCL/` with comparison plots against the historical results.
  • Checked-in thesis/publication material plus preserved source and data inventories.
  • Automated test coverage for the main solver families and historical variants.
