How do I delete a character when I am at the wiliki% prompt?

You should be able to either hit backspace or control-h, or the delete key. It all depends how things have been set up on your particular account.

You can determine which by typing the command "stty" at the prompt. It'll print out something like: 

speed 9600 baud; evenp hupcl susp = ^Z; dsusp = ^@
intr = ^C; erase = ^H; kill = ^U; start = ^Q; stop = ^S; swtch = ^@;
-inpck -istrip icrnl -ixany ixoff onlcr
iexten echo echoe echok
This seems like a mess, but the "erase = " on the second line tells you what key deletes the character at the cursor. In this case (on my machine) it is a control-h. If it has something like ^?, that's usually the delete key.

You can also delete an entire word with control-w.

What does "text file busy" mean?

This is an error message that shows up when you try to remove or replace a running program. For example, if your a.out is running while you try to recompile and relink your program, the linker will complain a.out: text file busy when it goes to replace your existing a.out.

This situation sometimes arises when you are working at home, you are running a.out, and your phone line drops. Sometimes the a.out just keeps running. Another place this happens is if you accidentally hit a control-Z when it is running. This stops the program and lets you do other things, but it doesn't actually "kill" the program, so the system thinks it is still running (that's so you can actually restart the program after you've stopped it).

The simplest way to fix the problem is to type:

ps -u name
where name is your account name. This lists all programs you have running, even those started from other terminals at other times. Look for a program named a.out (or whatever text file is said was busy). If you find it, type
kill number
where number is the value in the process id field in the output of ps.

How do I kill a program that won't stop?

The usual way to kill a running program is to hit the interrupt key (which is a control-C, hold down control and C at the same time). If that doesn't work for some reason, try hitting the kill key (which is a control-\).
