How can we read the times (10:31)? Is the colon a mistake?

No, the colon is not a mistake.

The way to read this is to take advantage of a feature of scanf we haven't talked about yet: if you put any characters other than spaces or formatting codes between the double quotes, scanf expects those characters to be present in the input. For example, if you write

scanf("%i,%i", &var1, &var2);
the scanfwill expect input to read two numbers with a comma in between them.

Do we always read exactly four different sets of times?

No, you should be able to read any number of different sets of times, where the input is terminated by the user entering a control-d.

Is the number of minutes for an average trip always an integer?

Yes. You can compute it simply by adding up the total number of minutes all of the trips take and dividing it by the number of trips. If you divide two integers, the result is an integer, and the fractional part of the answer is thrown away (e.g., 33/10 is 3). You need not worry about trying to round this value.

What does input "garbage" look like?

The term "garbage" means any kind of input you weren't looking for. So if the user types anything else that two pairs of colon-separated numbers (or EOF, when the user is done with input), that's garbage. That could be typing letters instead of numbers, failing to provide the colons, anything.

What do we do if there is more than one error in an input line?

First, if you can't read both input times correctly, you exit the program printing the "Couldn't read times" message (so even if you read the first time correctly, you don't bother checking it's validity).

Second, assuming you could read both input times correctly, you first validate the starting time. If there's an error, you print the "Invalid starting time" message, and don't bother to check the finishing time (so if the starting time and ending time are both invaild, you only write a message about the bad starting time).

Finally, assuming the starting time was valid, you should check the ending time. And if there's an error, you print the "Invalid ending time" message.

Do we have to prompt for input?

No. Do not prompt for input. When you run your program the input and output should look exactly like that on the handout for assignment #3.

Do we count the lines with errors when we print the average trip length?

If there are any errors at all in the input, you do not print the average trip length. That means that if any error occurs, you need to set a variable to remember that it happened, and the you have to check that variable before computing and printing the average.

Can we assume the height and width are odd numbers?

Yes and no.

The assignment description states that you can assume the height the user enters will be an odd number. However, you cannot assume that the width will be odd.

What that means, in practice, is that we will not test your program with an even height.

How do I compute the largest or smallest trip lengths?

See the answer to sample exam question number four for an example of how to find the largest number in the input. You can use variants on this technique to compute the largest and smallest trip lengths.

Can we assume that there will be only one pair of values entered (a single height and width)?

Yes, the user will only enter one pair of numbers (e.g., 5 7) each time they run the program.

What should our program do if the user enters garbage instead of the height and width?

Just have your program quietly quit. But we will only be testing your program with legitimate input values.

Can we assume the height and width are going to be five or larger?

Yes and no.

You can assume the height will be at least five, but the width can be as low as three. We will not be testing it with a smaller height or width.

In numbers1.c, do we read input for the height and width or define constants?

You need to read a single width and height for both numbers1.c and numbers2.c. The primary difference between the two files, is that numbers1.c just prints the 0, 1, and 7 with whatever size the user specified, while numbers2.c prints 0 through 9.

Can I make numbers1.c and numbers2.c repeatedly read input values?

We are going to test these programs by giving them a single pair of input values. So it's alright if you have an input-reading loop, but it's not required.

What exactly does it mean to use the functions in timeutils.c?

The idea is that you copy this file over to Wiliki. You can then write your main program (and any other functions you need) in the source file time1.c. When you are ready to make your executable, you compile time1.c and timeutils.c into object modules and then link them together.

At the end, you turn in only time1.c and we will automatically compile and link it with our own timeutils.c.

How do I use all of the functions in timeutils.c

That's the key part of this assignment. What you have to do is figure out how to call them, what parameters they expect, and what they return. The comments and the function header help a bit. For example, 

/* Compute the difference between start_hour_of_day:start_minute_of_hour
   and end_hour_of_day:end_minute_of_hour */

int timeDifference(int start_hour_of_day,
                   int start_minute_of_hour,
                   int end_hour_of_day,
                   int end_minute_of_hour)
The header says that this function takes four integers and returns an integer. The comment says what those four integers it takes and the integer it returns represents.

So now you can write a little main program to see what it does. The main program just consists of a bunch of calls to this function, passing it different values and printing its return value. 

/*
 * Try out the timeDifference function.
 */
#include <stdio.h>

main()
{
  int timeDifference(int start_hour_of_day,
                     int start_minute_of_hour,
                     int end_hour_of_day,
                     int end_minute_of_hour);

  printf("timeDifference(10,20,12,15) returns %i",
         timeDifference(10,20,12,15));
  printf("timeDifference(0,0,2,0) returns %i",
         timeDifference(0,0,2,0));
  printf("timeDifference(12,0,14,30) returns %i",
         timeDifference(12,0,14,30));
  return 0;
}

When you enter and run this little test program, you'll start to see what the timeDifference function does. You need to write a similar program to test each of the other functions we have provided for you, so that you can begin to understand those functions, as well.

Can we write our own functions?

Yes, and you are encouraged to do so, as you'll find it quite convenient to write some functions on top of the ones I provided. Just be careful that you don't write functions that replace the function you have been given. And do not attempt to redefine or change the functions we gave you.

Do we need write separate functions for each numbers (like makeOne, makeTwo, etc...)?

You don't have to define functions like these, but you'll find that doing so makes your program much easier to read and debug. The tradeoff is that all these functions will make your program somewhat longer.
