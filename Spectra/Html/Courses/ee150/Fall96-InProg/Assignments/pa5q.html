Questions about Functions

How do we use insertDoubleValue?

I just don't get it. How am I supposed to read the input?

You should use readValues. You give it an array and tell it how many values to read. It then tries to read that number of values, stopping if it encounters garbage or the end of the input.

It always returns the number of values read, or -1 if it encounters the end of the input (EOF). That means that if there is garbage in the input, its return value will be less than the number of values you asked it to read.

Do we care about the format of the input?

No. It's perfectly reasonable for the user to have any amount of blank space or blank lines between input values. For example, it is perfectly legal for the weights to look like this: 

10 20

30 30

10

The input in the example was formatted the way it was to make it easy to read, but the input does not have to be broken up into lines or to have a blank line before the scores.

How do we detect "missing" input values?

An input value is "missing" if you hit EOF when you go to read it. For example, if you try to read 5 weights and you find only 1 weight and then EOF, you are missing the remaining weights.

But assuming you read input using readValues, as far as program is concerned, an input value is missing if readValues returns something other than the number of values you were trying to read.

Do we assume all scores are integers?

As far as your program's processing is concerned the answer is "yes". This means you can declare integer arrays to hold the weights, the minimum scores for grades, and the scores you read in for a given student. If the user actually enters non-integer input, the readValues function will detect an error and return a value indicating that an error took place.

Is it an error if the user hits control-D as the very first input?

Yes. The result should be the error message about being unable to read the number of assignments.

Can the number of assignments be 0 (or even negative)?

No. You can assume the user will enter a positive number for the number of assignments. (It would be easy to add a check to ensure it is not negative or zero, but you don't have to, as we won't test your program with that input.)

What should the program do if there are too few weights, but the right number of grades?

I guess you are talking about input like this:

5 20 10 15 90 80 70 60
The thing is, in this case there are 5 weights in your input:
20 10 15 90 80
They just have to be spread out on two lines. Then there are two scores 70 60. So the error message, in this case, should be about the weights not summing to 100.

What do we do if the user types in "5falksj" for the number of assignments?

Well, 5 would be the number of assignments and should be successfully read in. But whatever we're reading after the number of assignments, such as the weights, will be read incorrectly (since there will be an error when we try to read the first weight).

For Part 4, is any answer beginning with "e" the same as "equal" and beginning with "s" the same as "standard"?

No, you need to read in the entire string. An input like "something" is not "standard" and should cause an error message to be printed.

Can the input be a real number?

No. If there's a real number, it's going to cause an error. For example, if the input was 6.6 10, then when you call readValues, it'll read the first 6 correctly and return a 1. The reason is that the . in .6 will cause it to fail to return the second number.

Do we stop reading input after we encounter an input error?

Yes. As soon as you detect an input error, you write an error message and terminate the program (without writing the summary information).

How do we read in the words for Part 4?

Start by trying to read in the weights with readValues. If it returns a 0, that means that it thinks there is garbage (non-numeric characters) on the line rather than numbers. In that case, you can read the characters into a character array with the getline function. That will read the entire line into the array.

This also works for reading in the grading scale.

For Part 2, do we have to store all the averages into an array?

No. The example I/O assumes that the input is in a file and the program is run with the input redirected. It then shows the output that would result from that input.

If you run the program and type in the input from the keyboard, you should see the average and the grade after each student's scores, just like in Part 1.

For Part 4, do we have to worry about upper- and lower-case characters for "equal" and "standard"?

You need only check for the lower-case versions of the words. Any uppercase (e.g., EQUAL) or mixed case (e.g., Equal) word will result in the error message about "unknown weights" or "unknown grading minimum".

For Part 4, do we have to worry about spaces before the words "equal" and "standard"?

No. I'm always going to place the word at the beginning of line for testing.

However, you should use getline to read in the word and then simply treat the whole line as the word (so leading or trailing spaces will cause the string comparisons with "equal" or "standard" to say the strings differ).

For Part 4, do we only read words for the weights and the grading scale or are we still supposed to handle numbers?

You have to handle both. You first try to read the input with readValues. If that fails, then you try to read the input wtih getline.

What do we do if the user doesn't enter any student scores?

If you encounter EOF when you go to read the first score, you do not write any kind of error message. You just execute your normal code (e.g., printing out the summary information). If you encounter garbage (readValues returns less than the number of assignments), then you write an error message about being unable to read the scores for student #1 (like you would for any other student).

What should our programs do if we hit EOF when we're reading the first student's scores?

Your program doesn't do anything special if there are 0 students. For Part 1, there's just no output. For the later parts, it'll write the "There are 0 students" message, but that's it. (The right behavior should just "fall out"; it doesn't require any extra work.)

For Part 3, I don't understand what we're suppposed to do if there are more students than there are supposed to be?

This question is asking what to do if there are, say 20 students, but the first line of the input promised only 10.

The idea is that your input reading loop stores the averages of the first 10 students into the array. When it reads the 11th, it prints the error message about "too many students", and leaves the input reading loop. You then do the normal processing, printing the averages you actually did compute and so on.

What should the error message be if we hit EOF in the middle of reading the weights?

It should be the message about "Can't read expected number of weights". It should not be "The sum of weights doesn't add to 100". Do not sum the weights until you are sure you have read the expected number of weights.

When do we produce the error message "Can't read expected number of weights"?

Conceptually you produce this message when you try to read the weights and you encounter EOF or garbage.

From a programming perspective, you care only about the return value of readValues. If it is not -1 and it's not how many values you expected, then you write an error message.

For part 3, when exactly do we produce the error message "Processing only XXX students"?

The first line of the input says how many students you expect to read. If you read more than that, you produce this error message, and ignore all remaining scores.

This means you need to keep count of the students you read and update it each time you successfully read a student.

You should treat this message like a warning and still print out the summary information. The catch is that you only produce the summary for the students you successfully read and stored in the array.

For parts 3 and 4, we no longer have to worry about the error message about there being more than 10 assignments, right?

Correct. You are now using malloc to allocate space for all the arrays that can vary in size, so whatever number of assignments the user types in is alright (we are assuming the user won't type in 0 or a negative number).

How do I determine the grade for a given weighted score?

We talked about this in class. To determine that 85.67 is a B, for example, you need to do several things. You should have an array containing the minimum scores for the various grades, where the first element is the minimum score for an A, the next element is a minimum score for a B, and so on (e.g., 90, 80, 70, etc...). You also need to have a table containing the grades themselves, where the first element is an 'A', the next is a 'B', and so on. You then take the weighted average and search the table to find the first value greater than the weighted average. The position of that element corresponds to the position in the table of grades that contains the desired grade.

How do I produce the summary information?

There are two parts to the summary information.

The first is a count of the number of students. That's easy--each time you successfully call readValues to read one student's scores, you need to update a counter.

The second is the counts of each grade. The ideal way to do this is to have an array of counters. You then update counters[0] when you assign a student an A, counters[1] when you assign a student a B, and so on.

Can we assume the minimum scores are 90, 80, 70, 60, and 0?

Absolutely not. You are reading those values in, so they could be whatever values the user entered (although you may assume that they are entered in descending order). You should read them into an array and then search the array in order to assign the scores.

How come when I print the weighted average it comes out 68.00 instead of 68.60?

The best guess is that you calculate the average using integer division. That is, it is likely you did: 

long sum;
double result;

result = sum/100;
This gives the wrong result it's going to do integer division on the sum. Instead you need to do either: 
result = (double) sum / 100;
or 
result = sum / 100.0;
Either of these cause the division to be done using reals, rather than integers.

When I read "equal" for the weights, how do I calculate the values to put in the table of weights?

The correct thing to do is to use integer division to divide 100 by the total number of assignments and put the resulting value into the weights array. Make sure you then add the weights together to see if they sum to 100.

For example, if you have three assignments, put 100/3, or 33 into each weight. When you sum them together, it comes out to 99, which should result in "Weights do not sum to 100." error message.

How do I return an array from a function?

You can't return an array from a function. You need to pass the array as a parameter and then the function can fill the array. This is what is done by readValues, so you can use it as a model.

What does the error message "Pointers not compatible" mean when I'm passing array parameters?

It most likely means that you are trying to pass an array of one type to a function that expects an array of another type (e.g., passing readValues an array of doubles when readValues expects an array of ints.

How do we use insertDoubleValue (for part III)?

See Chapter 8 of the textbook, page 159. The function table_insert in that program is very similar to insertDoubleValue, the primary difference being that insertDoubleValue works with doubles rather than ints.

In addition, I've revised the tables.c on the web so that it now describes the insertDoubleValue's parameters in more detail.

The main idea is that to use table_insert in the book, you need to pass it an array, a counter that says how many items are in the array, and the value you want to insert. insertDoubleValue is exactly the same. You have to pass it an array of doubles, a counter of the elements you've put in the array so far (so this has to start at 0, and then is incremented after you put each value in the array), and the particular value you want to place in the array (which for this problem is the average score you've computed).
