What To Do

Part 1: Create a file named scores1.c. This program will read in grading information (weights and the grading scale) and individual student test scores, and will print the weighted average for each student, along with the students grade.

The input is of the following form:

  • The number of assignments in the course so far.

  • The weights (a number between 0 and 100) for each assignment. A weight of 40, for example, means that assignment is 40% of the course grade. There should be as many weights as there are assignments.

  • The minimum score necessary for each grade (A, B, C, and D). That means there are 4 minimums. The minimum for an F is automatically assumed to be 0 and is not provided as input.

  • The scores for each student, where each student will have the same number of scores as there are assignments and weights.

Here is some sample input and output.

5
20 40 10 5 25
90 80 70 60

100 100 100 100 100
100.00 A
80 82 84 81 87
83.00 B
90 95 85 70 100
93.00 A
50 0 75 50 60
35.00 F
67 75 58 62 66
68.80 D
85 70 60 65 80
74.25 C

The first line says there are 5 assignments. The second line says the first assignment is worth 20%, the second assignment is worth 40%, the third assignment is worth 10%, the fourth assignment is worth 5%, and the last assignment is worth 25%. The third line says that the course is using the traditional grading scale: 9)% for an A, 80% for a B, 70% for a C, and 60% for a D. The remaining lines are the scores for each student: all 100's for the first student; 80, 82, 84, 81, and 87 for the next student, and so on.

The output follows each students scores. The first student got a 100.00 weighted average, which is an A, according to the provided grading scale. The next student got a weighted average of 83, which is computed by calculating the weighted sum (20*80 + 40*82 + 10*84 + 5*81 + 25*87) and dividing it by 100.

For this part, you can assume the user will be well-behaved. That is, you may assume that the input will be in an appropriate format (i.e., there will be no garbage in the input, you will have a reasonable number of assignments, the weights will add up to 100, there will be the right number of scores for each student, and so on.) You can assume the user will enter no more than 10 for the number of assignments, but that there may be an infinite class size (e.g, you don't know how many different student scores will be entered).

To help you implement this program, we have provided a source file, tables.c, which contains a collection of potentially useful functions. For this assignment, you should take advantage of readValues, which reads n values into an array and returns the number of items it successfully read. For debugging, we have provided a function printValues, which prints the first n values in an array. As with the previous assignments, you will compile this source file, then compile your program, and then finally link the resulting object modules together to form an executable. As a result, you get to pretend the functions in this file are part of the language, like printf and scanf.

This time, in addition, we have also provided a header file, tables.h, which contains the prototypes for the various functions. You should use copy this file over to your machine and then use 

#include "tables.h"
to include the prototypes it contains.

You must use separate compilation, and simply link these files in with your source. Do not copy the functions into your source file. That's because the grading program will automatically try to link the functions in tables.c in with your program.

Part 2: This part of the program is to extend your program to ensure that it behaves sensibly even if the user provides invalid input and to provide some additional summary information after all the input has been processed. Before starting this part, copy your solution to Part 1 into a file named scores2.c.

There are a variety of mistakes the user can make in providing the input to this program. Your program's job is to detect these mistakes and if it finds any of them, it should print an error message and stop. Here are the list of things to check for and the error message to print. You should check for them in this order.

  • Garbage or EOF when trying to read the number of assignments. 

    Can't read the number of assignments.
  • A number of assignments greater than the maximum allowed number of assignments (10 for this program). 

    Too many assignments (X).  Can only handle 10.
    The X in parentheses should be replaced with the actual number provided.

  • Fewer weights provided than assignments. 
    Can't read expected number of weights.
  • Weights do not add up to 100. 
    Weights sum to X rather than 100.
    The X in this output should be replaced with what the weights sum to.

  • Not being able to read the minimum scores for each grade. 
    Can't read minimum scores for grades.
  • A problem reading the scores for a given student. When you go to read those scores, you should either hit end of file or read all the scores you expect (one for each assignment). 
    Can't read scores for student X.
    The X in this output should be replaced by a count of students read so far (1 for the first student, 2 for the second student, and so on).

In addition, you will also extend the program to produce some summary information after the last student is read. (This is only produced if there are no errors.) The summary information appears after the last student's average and grade are printed, and includes the number of students getting each grade (unless there are no students with that grade, in which neither the grade nor the number of students is printed). Here's some example input: 
5
20 40 10 5 25
90 80 70 60

100 100 100 100 100
100  90  92  95  87
90   95  85  70 100
90   90  93  89 100
67   75  58  62  66
85   70  60  65  80
Here's the output when we place this input in a file and run our program with its input redirected to come from that file. 
100.00 A
91.70 A
93.00 A
92.75 A
68.80 D
74.25 C

There are 6 students.
A: 4
C: 1
D: 1

Part 3: This part is to extend the grading program to print the averages in sorted order, as well as to handle any number of assignments for each student. Before doing this part, copy your solution to the previous part into scores3.c.

In particular, the input now not only specifies how many assignments there are, but how many students. The program now dynamically allocates the arrays that hold the weights and each student's scores. In addition, the program requires a dynamically allocated array of doubles to hold each students average.

Here is some example input: 

5 10
20 40 10 5 25
90 80 70 60

100 100 100 100 100
100  90  92  95  87
90   95  85  70 100
90   90  93  89 100
67   75  58  62  66
85   70  60  65  80
For this input, the program's output is: 
100.00 A
93.00 A
92.75 A
91.70 A
74.25 C
68.80 D

There are 6 students.
A: 4
C: 1
D: 1

Notice that the input is printed in sorted order, with the highest scores first, and the lowest scores last.

In addition, notice that the first line of input says there will be 5 assignments, and 10 students. If more than 10 students are provided in the input, print a message before the output above that states 

Only first X students processed.
where the X is replaced with the expected number of students.

One last change is that now if there is a problem reading in the number of assignments or the number of students, the program should print this error message. 

Can't read the number of assignments and students.

The tables.c file provides a function to insert a value in the correct place in a table of doubles, so you should use that to help construct the needed sorted array of averages.

Part 4: This part is to let the user use text as an alternate way to describe the weights and grading, and to print the minimum and maximum scores. Before doing this part, copy your solution to the previous part into scores4.c.

The first change is to allow the user to use "equal" as an optional way to specify that each assignment should be equally weighted. That is, instead of entering the weights, the user just puts the word equal on a line by itself. If there are 5 assignments, this results in each assignment having a weight of 20. If there's no way to have equal integer weights, the program can simply print the message about the weights summing to something other than 100. If the user enters anything else, the program should print 

Unknown weight request: xxxxxx.
where the xxxxx is replaced with the string the user actually entered. (Of course, the user can alternatively enter the weights with values, just like before.)

The other change is to allow the user to use "standard" as an optional way to specify the grading scale, just like "equal" for weights. With "standard", an A is 90, a B is 80, a C is 70, and a D is 60. There should be similar error message to the one above if there's a problem, but it should print 

Unknown grade request: xxxxxx.

The final task is to print the highest and lowest scores.

Here's input that takes advantage of the newly added messages. 

5 10
equal
standard

100 100 100 100 100
100  90  92  95  87
90   95  85  70 100
90   90  93  89 100
67   75  58  62  66
85   70  60  65  80
And here's the resulting output. 
100.00 A
92.80 A
92.40 A
88.00 B
72.00 C
65.60 D

There are 6 students.
A: 3
B: 1
C: 1
D: 1

Highest: 100.00
Lowest: 65.60
