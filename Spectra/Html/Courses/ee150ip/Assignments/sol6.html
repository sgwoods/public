Programming Assignment #6
Solution


Part 1:
The idea with this program is to start by breaking it into two parts: a function, readCodes, to read the codes and set up "before" and "after" arrays, and a loop that reads lines, uses another function, convert, to compute the transformation and store it into a new array, and a printf to dump that array.

readCodes reads the line with the transformations, finds the blank separating them, and then copies the appropriate pieces to the "before" and "after" arrays. convert takes the input array, the "before" and "after" arrays, and an output array. It looks up each character in the input array in the "before" array and then stores the corresponding character from the "after" array into the output array.

Our solution is in convert1.c.

Part 2:
The main program for this one is the same as before, as is the convert function. The difference is that readCodes has to make an extra pass through the "before" and "after" arrays to expand the ranges they contain. This is done by calling a new function, expand, on each of those arrays. expand is basically a state machine that does different things depending on whether the last character was a dash. If it isn't, it just copies the character into a new array (as long as the character is not a dash). If it is, it puts all the characters represented by the range into the new array.

Our solution is in convert2.c.

Part 3:
The key to this program is using all functions you were given to work with. The first part of the program is a loop that uses readStudentInfo to read each group containing a name, account, and id and then store it in the next available entry in the table. At this point, the program indicates that no students have averages yet by setting their average to -1.

The second part is a loop that uses readStudentScores to read IDs and scores, lookupStudent to find the table entry with the provided ID, computeAverage to compute the student's average, and a simple assignment to save the average.

The last part is a loop to print all the students in the table, along with their average (unless it's still -1, in which case it knows that they never had any scores read in.

Our solution is in grades1.c.

Part 4:
Here, the primary difference is in the second part of the program. The scores are stored in the next row of a two-dimensional array, and the number of that row is stored in the score_index field of the structure containing that student's ID and name.

The other change is that when the program does to print the scores, it uses the score_index to tell it which row of the array contains those scores, and then it prints that row.

Our solution is in grades2.c.
