What To Do

Part 1: Create a file named convert1.c. This program transforms its input according to a tranformation scheme defined on the first line of input. To explain this, here's some example input and output:

abcde bcead
alex was here
bldx wbs hdrd
a better example
b cdttdr dxbmpld

The first line of the input consists of two parts: a list of characters and then a list of replacement characters. The idea is that we replace every instance of the i'th character in the initial list with the i'th character in the replacement list. In this case, that means replace every "a" with a "b", every "b" with a "c", every "c" with an "e", every "d" with an "a", and every "e" with a "d".

The program then reads the remaining lines of input and then writes what they look like after the substitutions have been made. So "alex was here" becomes "bldx wbs hdrd", after substituting for the "a" and the "e". When no substitution is defined for a character, the program just passes it through to the output unchanged. Similarly, "a better example" gets transformed into "b cdttdr dxbmpld".

So essentially your task is to write a program that reads the transformation information and then enters a loop that reads each input line, transforms it, and outputs the result.

Why write a program like this? Because it's a simple version of an encryption program. Encrypting a file encodes it in such a way that it is unreadable unless the person has a code to decrypt it. In this case, we are using a very simple (and easy to break) method of encryption. If we wanted to, once we have transformed the input to output, we can get the original back by specifying the reverse transformation (giving a first line of "bcead abcde" and the input "bldx wbs hdrd" will give us "alex was here" as the output).

In this part, you also need to check for certain errors. Here are examples of those errors and the error messages you need to produce (any error is fatal).

  • EOF when trying to read the conversion tables.
    ^D
    Missing conversion tables.
  • No blank on the first line (or an immediate return character).
    whatever
    Missing blank.
  • Different numbers of text characters and replacement text.
    alex whatever
    Can't map "alex" to "whatever".
    This can also happen if the second is longer than the first:
    whatever alex
    Can't map "whatever" to "alex".

Any characters in the input that are not in the transformation list are passed through to the output untransformed. (And although we haven't shown it, the transformations can contain any characters other than a blank, including upper and lower case characters, punctuation, and so on.)

To help you implement this program, we have provided a source file, helpers.c, which contains a collection of potentially useful functions. We have also provided a header file, helpers.h, which contains the prototypes for the various functions. You should use copy this file over to your machine and then use 

#include "helpers.h"
to include the prototypes it contains.

You should write this program as a series of small functions. It has been deliberately designed to be extremely difficult if you do not try to write it as a series of small functions or if you avoid the functions we have already provided for you. This will be especially true when you tackle Part 2. If you try to write it as a single lengthy main program, our estimate is that you can expect to spend double or triple the 5-6 hours it will take to write if you use the functions we have provided for you and you carefully build the program from small pieces.

You must use separate compilation, and simply link these files in with your source. Do not copy the functions into your source file, and do not include helpers.c in your source file. That's because the grading program will automatically try to link the functions in helpers.c in with your program.

Part 2: The second part is to allow the user to specify ranges as part of the transformation description on the first line and do some additional error checking. This program is called convert2.c.

A range is a letter, a dash, and another letter. For example, "a-d" is a range representing the letters "abcd" and "a-z" is a range representing the entire alphabet. The idea is that the user can now specify transformations like this

a-z n-za-m
as a shortcut over specifying every letter in the transformations.
abcdefghijklmnopqrstuvwxyz nopqrstuvwxyzabcdefghijklm
The lengthier version is called the "expanded" character range.

Here is some sample input and output:

a-z n-za-m
alex was here
nyrk jnf urer
ee150 is great
rr150 vf terng
nyrk jnf urer
alex was here
This particular tranformation has the effect of rotating all the characters in the input by 13. What's neat about it is that you can apply the transformation once to get the scrambled version of your initial input, then apply it again to restore the original.

In adding the "ranges", you now have a new set of errors to check for. These all assume that you received two input items on the first line. If you only received one (or none), the error message will say there's no blank and you won't check for problems with dashes.

  • A dash at the start or end of the line.
    -alex whatever
    Dash at start or finish in "-alex".
  • More than one dash in a row.
    abbcd d--m
    Back to back dashes in "d--m".
  • The second character in the range coming before the first.
    d-a d-m
    Illegal range in "d-a".
  • The same character showing up more than once in the transformation (including characters that appear in multiple ranges).
    a-db-f a-m
    Duplicate letters in "a-db-f".

The very last error to check for is whether or not there are the same number of characters in the resulting expanded transformation (this is the "can't map..." error message from Part 1).

The first line of the input is the number of assignments. The next part of the input consists of three lines for each student. The first line is their e-mail account, the second line is their name, and the last line is their id. This part of the input is terminated with a period on a line by itself. After the period is another set of lines for each student, where the first line is the student id, and the next set of lines (one per assignment) is the student's score on the various assignments.

You need to check for the following errors:

    Can't read number of assignments.
  • Any sort of problem reading in the initial name, account name, and id. 
    Problem in reading student information into table.
  • Reading in an id and scores when there's no student with that id.

These functions include things such as reading in all the information for a student, printing an array of students in a form suitable for debugging, sorted an array of students by average, computing the average of an array, reading a score that's on a line by itself, and more.

Part 4: This program will extend the previous part to also print the scores for each student, along with the average. You should start by copying grades1.c into a grades2.c. The input to this part is the same, but the output differs. For the previous sample input, the output should be: 

Vinod Malhotra (vinod) 100
  [100,100,100]
Greg Uehara (guehara) 88
  [90,90,85]
Rahul Chattergy (rahul) 80
  [90,80,70]
Tony Kuh (kuh) 80
  [70,80,90]
Shu Lin (lin) - no input scores
James Yee (yee) - no input scores

The only difference is that each student's scores follow that student. The catch is that you are not allowed to modify the definition of a struct student to do this assignment.

The way your program should work is now read each set of student scores into a row within the two-dimensional array. The scores for the first student go into the first row, the second for the second student go into the second row, and so on. You now must store the row number of the array where you put a student's scores in the structure representing that student (but not the scores themselves). We have conveniently added a new field to the struct student to hold this value. (We have also added some new helpful functions to student.h and student.c,, so you should retrieve the latest versions of the files.)

Your program behaves substantially the same way as before, except that after sorting the scores by average, you need to print not only the information in the student structure, but also the scores stored in the row corresponding to that student.
