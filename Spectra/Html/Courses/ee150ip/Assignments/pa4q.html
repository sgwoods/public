I'm having a hard time getting started with the base1 program. What should I do?

You should attack this problem the way I have been attacking all problems in class. Pick a small piece of the problem and get that working and then gradually extend it from there.

So, for this program, one way to get started is to simply read in the operators and print out an appropriate message for each operator as you read it (like "PLUS" for "+", and so on). That is accomplished almost exactly like the checkbook example in class.

Once you have that working, imagine writing the program using scanf to read the two values (as if they were already in decimal). Given that, you can fill in the actual calculations the program would do (adding the inputs, subtracting them, and so on) and printing the result.

Then try to extend the program to print the result not only as a decimal number, but also as a binary number, using the putNumInBase function.

Now, finally, worry about replacing scanf with a couple of calls to getNumInBase to read binary values as input rather than decimals.

I'm having a hard time getting started with the "primes" program. What should I do?

The first thing you need to do is write a program that can read in a single value and determine whether it is prime. This divides into two parts: reading the value (which we can do with scanf) and then calling a function to determine whether it is prime (so we have to pass the input value to that function).

The catch is that function doesn't exist. However, see the answer to the question about how to determine whether a number is prime for some guidelines on writing that function. The function should return a 0 if the number isn't prime and some other value otherwise.

Once you have the function, you should be able to extend your program to read in a pair of values and then use a loop to run from one value to the another, calling your function on each value of the loop index, and printing the loop index when the function determines the number is prime.

How do I figure out what getNumInBase and those other functions in helpers.c actually do?

The way to find out what these functions do is to write a little main program that uses them. As in the last assignment, there are a few examples of calling these functions in the assignment description. Write a main program that includes these calls and see what they do with various inputs. For example, 

#include <stdio.h>
main()
{
  long getNumInBase(int base, char follow);
  long n;
 
  n = getNumInBase(2, ' ');
  printf("%li", n);
  n = getNumInBase(2, '\n');
  printf("%li", n);
}

Try running the program with different inputs to see what it prints for n. This should give you an idea of what the function does and how to use it. You then need to do something similar with the putNumInBase functions.

How come I get an error about pow and floor not being defined when I compile helpers.c?

The functions in helpers.c use the math library. To use the math library functions, you need to add -lm (that's a minus sign, and ell, and an m) to the end of the command line use use to link the object modules together, as in:

cc base1.o helpers.o -lm

Do I need to copy the functions in helpers.c into base1.c and base2.c?

Absolutely not. In fact, you will get lots of points off if you do. Write base1.c using the functions in helpers.c, use separate compilation to make the executable, and then turn in only base1.c and not helpers.c. We will automatically compile what you turn in with our own version of helpers.c, which will result in "doubly defined symbol" error messages if you have copied those functions into the base1.c.

How come sqrt is giving me the wrong result?

sqrt is a function that expects to be passed a double. If you pass it an int, there are going to be problems--unless you make sure to do two things. The first is include the math.h header file. This includes a function prototype for sqrt. The second is to compile with the -Aa option. This invokves an ANSI-C compiler. The ANSI-C compiler reads the prototype, realizes that you are passing an int to a function expecting a double, and automatically converts the int to a double for you.

How should we handle EOF and input errors in the prime program?

Basically anywhere you read input, you could potentially read garbage or an EOF. So those places are the question about whether the program should print any primes, the question about whether the numbers should be printed in ascending or descending order, and where you read the range of values.

For the questions, it's reasonable to treat EOF as a "NO" answer. If you do this, your program will automatically quit when you ask the user whether there any prime numbers to print. Similarly, the "yesorno" we did in class treats junk input as a "NO", so your program will quit if the user enteres garbage in response to that question.

So what about the other question? If the user enters EOF to that question (about ascending/descending order), it will be treated as a no, and your program will go ahead and print the primes and then ask the question about whether to print more primes. Since the EOF is still there, you'll get another "no" and the program will quit. Similarly, if the user enters garbage, that will be treated as a "no" to the ascending/descending question (with the remainder of the line ignored). So the program will then ask the question about continuing, at which point the user can point in a new answer.

Finally, we have the two numbers the user enters for the input range. Here, you have to test the result of the scanf you are using to read them in and if it's not two (assuming you are reading both values with a single scanf), you need to exit the loop. The easiest way to accomplish this is with a "break;" in the "if". A "break;" leaves the nearest enclosing loop or switch. That means if your scanf reading is inside a while loop, the "break;" will kick you out of that loop.

How do I interpret the output for the binary arithmetic program?

The idea is that you input an operator and a pair of binary numbers. The output is the result of applying the operator to two binary numbers. For example, take the first input/output pair in the handout:

+ 1001 1101
1001 (9) + 1101 (13) = 10110 (22)

Here we are requesting that it print the result of adding the binary number 1001 to the binary number 1101. The output shows the result of that addition in binary, along with the base 10 numbers each of those binary numbers represent (in parentheses following the number). So 1001 is 9 in base 10 (you can figure this out by hand by taking 1*2^0 + 0*2^1 + 0*2^2 + 1*2^3, but getNumInBase will return 9 when it reads in this value, so you don't have to).

How come I get "log10: Domain Error" when I output the result of a subtraction with putNumInBase?

The problem is that putNumInBase doesn't know how to display negative numbers (and that error message is C's unfriendly way of telling you that). What you have to do to get around this problem is check whether you have a negative result, and if you do, write the negative sign yourself and then pass putNumInBase the absolute value of the number.

How can we tell if a number is prime?

The definition of a prime number is a number that is divisible evenly (with a remainder of 0) only by 1 and itself. To tell if a given number is prime, you therefore need to divide it by 2, then by 3, and so on, all the way up to one less than that number. If there's any division that has a remainder of 0, it's not prime.

Now, there are actually one big shortcut. One is that you don't actually have to go all the way up to one less than the number. You can go up to the square root of the number (that's because if there's a factor of the number that's greater than the square root, there is also one that's less that the square root, so we would have already detected that the number isn't prime).

Of course, the question is going to come up: How do we tell if the remainder when we divide two numbers is zero? And the answer is, you just compute the remainder directly, using the % operator and compare that result with zero.

Is 1 a prime number?

Yes. It is divisible evenly only by 1 and itself.

How do we use a switch when we want to do more than one action for a given case?

You can have a case in a switch have more than one action simply by listing the actions after the case label. Here's a template: 

switch( VARIABLE )
{
  case VALUE-1:
    ACTION-1
    ACTION-2
      ...
    break;
  case VALUE-2:
    ACTION-1
    ACTION-2
      ...
    break;
  ...
}
The idea is that the break kicks you out of the matching case, so you don't need any curly braces around the actions you want to do.
