You can assume that the user will be well-behaved, that the user will always enter the input in the form of an operator, some number of spaces, a binary number, some more spaces, another binary number, and a carriage return. the times will always be valid, and that the second time will always be after the first.

You should build this program based on the functions in the file helpers.c. The idea is that you will compile this source file, then compile your program, and finally link the resulting object modules together to form an executabl. As a result, you get to pretend the functions in this file are part of the language, like printf and scanf.

You will find the following functions quite useful: 

long getNumInBase(int base, char follow)

getNumInBase takes a desired base (such as 2 for binary) and the character we expect to follow the number. It reads a number entered in that base and returns its value in base 10. The number is allowed to be preceded by an arbitrary number of spaces. The function returns the value in that base or a negative number if there's any kind of error (such as a digit not in the base, or the number not terminated by the follow character). For example, given this input, 

1111   11101
the following call: 
n = getNumInBase(2, ' ');
will place 15 (the base 2 equivalent of 1111) into n, reading those characters. The blank tells it to verify that the 1111 is in fact followed by a blank and to read than blank. 
void putNumInBase(long value, int base);

putNumInBase takes a value and a base to write it in, and displays that value in that base. So the call 

putNumInBase(15, 2);
prints 1111 (the base 2 equivalent of 15).

Some of the other functions in the helpers.c file may prove useful as well. However, you'll need to examine the source to determine exactly what else you can use from that source file.

You must use separate compilation, and simply link these files in with your source. Do not copy the functions into your source file. That's because the grading program will automatically try to link the functions in helpers.c in with your program.

To do this part, you need to understand the various return values from getNumInBase. They are:

  • -1 if it hits something other than an appropriate digit for the base it is trying to read.
  • -2 if the number is missing (it hits a return rather than the number).
  • -3 if we hit a newline after the number but expected something else.
  • -4 if we didn't hit a newline when we expected one
